Task Runner

Sassのコンパイルやhtmlのバリデーション、ブラウザのリロードなど、
細々した作業は自動化する。

Grunt [ぐらんと]

Documentation

  • NodeJSアプリケーション
  • 豊富なプラグイン
  • 多くの日本語情報

Gulp [がるぷ]

Documentation

  • 後発のタスクランナー
  • NodeJSアプリケーション

How to use

Required files

  • package.json
    使いたいパッケージ（Nodeアプリケーション）を記載
  • Gruntfile.js or gulpfile.js
    コンフィグ（タスクの定義等）

Start task

  1. $npm install
    package.jsonに記載さているパッケージをインストール
  2. $grunt(gulp) example
    grunt(gulp)のみだとdefaltタスクを実行

Sass [さす]

Documentation

Syntactically Awesome Stylesheets
  • Rubyアプリケーション
  • sass記法とscss記法があるが、通常scss記法を使用する
  • compassなど便利なフレームワークもあるが、最近失速気味

  • オフィシャルページの眼鏡に、コロン（ : ）とセミコロン（ ; ）がついている

Nesting

SCSS :
.example {
    background: black;
    .text {
        color: red;
    }
}
CSS :
.example {
    background: black;
}
.example .text {
    color: red;
}

@extend

SCSS :
%text_large {
    font-size: 1.3rem;
}
.text01 {
    @extend %text_large;
    color: red;
}
.text02 {
    @extend %text_large;
    color: blue;
}
CSS :
.text01,
.text02 {
    font-size: 1.3rem;
}
.text01 {
    color: red;
}
.text02 {
    color: blue;
}

@mixin

SCSS :
@mixin position($top, $right, $bottom, $left) {
    position: absolute;
    @if $top    != '' { top: $top; }
    @if $bottom != '' { bottom: $bottom; }
    @if $left   != '' { left: $left; }
    @if $right  != '' { right: $right; }
}

.box {
    @include position(50%, '', '', 20px);
}
CSS :
.box {
    position: absolute;
    top: 50%;
    left: 20px;
}

etc...

SCSS :
// Set Banner
$bannerType: campaign;
$keyColor: red;
$borderSize: 5px;

/* Banner */
.#{$bannerType}_banner {
    width: 250px - ($borderSize * 2);
    border: solid $borderSize $keyColor;
    a {
        display: block;
        padding: 10px 20px;
        background: $keyColor;
        color: white;
        &:hover {
            background: $keyColor * .7;
        }
    }
}
CSS :
/* Banner */
.campaign_banner {
    width: 240px;
    border: solid 5px red;
}
.campaign_banner a {
    display: block;
    padding: 10px 20px;
    background: red;
    color: white;
}
.campaign_banner a:hover {
    background: #c40000;
}

CSS Architects

cssは自由かつ簡単に記述しする事ができる為、肥大化しやすく破綻してしまいやすい。
その為、ルールを作り秩序をもたらす事が必要となる。

OOCSS [おー・おー・しーえすえす]

Documentation

Object Oriented Cascading Style Sheets
  • 2009年にNicole Sullivanによって提唱
  • 構造とスキンに分離してクラスを定義する
  • Object Oriented CSS by Nicole Sullivan

SMACSS [すまっくす]

Documentation

Scalable and ModularArchitecture for CSS
  • 2011年にJonathan Snookによって提唱
  • Base、Layout、Module、State、Themeの５つのカテゴリに分類しクラスを定義する
  • Your CSS is a Mess by Jonathan Snook

BEM [べむ]

Documentation

Block Element Modifier
  • 2012年にYandex社によって提唱
  • Block、Element、Modifierに分類し、厳格な命名規則に従いクラスを定義する
  • BEM. What you can borrow from Yandex
